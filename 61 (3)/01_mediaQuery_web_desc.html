background 정리

background : 축약형 && 멀티형태 호출가능
background로 호출 할 경우
장점: background 기능중에서 호출할 대상이 무엇인지 모를 경우, 호출가능 함
단점: background의 모든 기능을 한꺼번에 호출하기 때문에 매우 느려질 수 있음
background의 종류
background-color : 배경색상 (배경이미지나 비디오 대체할 때 사용)
tip) 모자란 영역을 채울 때 사용하면 매우좋음
background-image : 배경이미지(영역에 칠하는 개념)
tip) 모자란 영역까지 칠하기 때문에 이미지가 반복되어서 보일수 있음
background-repeat : 배경이미지 반복제어
tip) 초기화 부분에서 반복제어를 하지 않음, 반복이 필요할 수 있음
background-position : 배경이미지 위치조정
tip) left, top, center, 20px, 50% 명칭 및 수치적용 가능
background-size : html5에서 사용가능 함
+ 표준 브라우저에서만 사용이 가능함
+ 익스플로러 9버전에서는 사용이 불가능할 수 있음
+ 익스플로러는 10버전 이상부터 사용해야 css3의 기능을 사용할 수 있음
+ 구형 브라우저에서는 이미지가 원래상태로 돌아가기 때문에 사용이 불편함
+ 영역에 맞춰서 이미지를 제작하는 것을 권장함!
background-attachment : background-image를 영역에 붙일 때 사용
주의) 일부 모바일 기기에서 표현되지 않음
background-origin : 속성에 따라서 시작하는 지점이 다름( 속성의 left, top에서 부터 시작됨 )
background-clip : 속성에 따라 어디까지 표현할지 적용가능!

웹을 읽는 순서_속성(기본, id, class, tag)

  1. 가장 먼저 읽어주는 값은 tag
  2. 웹에서 읽어주는 속성중 빠른 순서는? id가 빠름
  3. 모든 영역에서 빠름, style 뿐 아니라 script에서도 id는 한번만 사용하기 때문에 매우 빠름!
  4. class와 tag중에 빠른 순서는? tag가 조금더 빠름
  5. class는 같은 이름을 여러번 사용할 수 있고, 한 객체에 여러개의 class명을 작성할 수 있음

축약형 표현방법들

축약형 표현방법들은 빠르지 않지만 코드를 정리할때 편리함

  1. font:italic bold 12px/1.5em gulim, arial, sans-serif
  2. font-weight의 경우 축약형 표현안에서 항상 위치가 바뀔수 있기 때문에 주의해야 한다.
  3. font관련 축약형은 표현 방법중 가장 가볍기 때문에 많이 사용함
  4. body영역에 제일 많이 사용하는 것이 font 축약형
  1. border:1px solid red ,축약형
  2. border-left, border-top, border-bottom, border-right , 위치값은 순서 상관없이 호출가능
  3. margin, padding : top-right-bottom-left 시계방향 회전
장면전화 : transition:width 3s linear 0.2s
transition-property : 속성변화
transition-duration : 장면이 전환되는 시간
transition-timing-function : 속도표현
transition-delay : 작동될 때 까지의 지연시간

form과 table

  • form은 action과 method의 속성을 가짐
  • action 줄을 통해서 db를 연동할 수 있음
  • table은 form 의 action이 없기 때문에 스스로 db나 control 페이지로 넘겨줄 수 없음
  • table은 주로 표 만들때 사용
  • fieldset은 form안에서 주제별로 묶어 줄때 사용
  • fieldset의 제목은 legend 작성
  • table의 제목은 caption태그 작성
  • table에서 어떻게 사용하는지 속성을 작성하는 summary가 있음
  • validator 확인해보면 summary 오류 발생, title로 대체가능함
